HTTP stavový kód = informace o proběhnutí přenosu dat

Dle HTTP stavového kódu dokážeme určit, zdali vše proběhlo v pořádku, nebo kde je chyba. Mezi nejčastější patří 200 = Vše OK a 404 = Nenalezeno

Typy kódů

Máme celkem 5 typů Informační (1xx), Úspěch (2xx), Přesměrování (3xx), Chyba uživatele (4xx) a Chyba serveru (5xx).

Příklady aplikací API

Informační (1xx)

  • 100 = Pokračovat
  • 101 = Přepnout protokol

Úspěch (2xx)

  • 200 = Vše v pořádku
  • 201 = Vytvořeno
  • 202 = Přijato
  • 203 = Neoprávněné informace
  • 204 = Žádný obsah
  • 205 = Obnovit obsah
  • 206 = Částečný obsah

Přesměrování

  • 300 = Více možností
  • 301 = Trvalé přesměrování
  • 302 = Nalezeno
  • 303 = Viz další
  • 304 = Beze změny
  • 305 = Použijte proxy
  • 306 = Rezervováno, ale neopoužito
  • 307 = Dočasné přesměrování

Chyba serveru

  • 500 = Interní chyba serveru
  • 501 = Není implementováno
  • 502 = Bad Gateway
  • 503 = Služba není k dispozici
  • 504 = Časový limit brány vypršel
  • 505 = Verze HTTP není podporována
  • 509 = Byl překročen limit šířky pásma

Chyba uživatele

  • 400 = Špatný požadavek
  • 401 = Neoprávněné připojení
  • 402 = Požadovaná platba
  • 403 = Zakázáno
  • 404 = Nenalezeno
  • 405 = Metoda není povolena
  • 406 = Nepřijatelné
  • 407 = Vyžaduje se ověření proxy
  • 408 = Časový limit požadavku vypršel
  • 409 = Konflikt na síti
  • 410 = Data jsou pryč
  • 411 = Požadovaná délka neodpovídá
  • 412 = Předpoklad se nezdařil
  • 413 = Žádost o entitu je příliš velká
  • 414 = Žádost-URI je příliš dlouhý
  • 415 = Nepodporovaný typ média
  • 416 = Požadovaný rozsah není uspokojivý
  • 417 = Očekávání se nezdařilo
